# Научный клубок

MVP GraphRAG-система для поиска по научно-техническому корпусу горно-металлургической отрасли. Система отвечает на вопросы пользователя на естественном языке и подкрепляет ответ ссылками на документы-источники.

## Что делает система

Система превращает вопрос пользователя в ответ с проверяемой доказательной базой:

- находит релевантные фрагменты в корпусе документов;
- показывает файл-источник, страницу и найденный фрагмент;
- формирует итоговый ответ по найденному контексту;
- поддерживает поиск между русским и английским языками;
- использует граф Neo4j для навигации по документам, фрагментам и метаданным.

## Корпус данных

В рамках MVP подготовлен корпус чистых текстов из **1 550 файлов**. Общий объем корпуса - около **48 млн слов**.

Корпус подготовлен на основе исходных документов в форматах:

- PDF;
- DOCX;
- XLS;
- PPTX;
- изображения и сканы;
- другие технические форматы.

Языковой состав корпуса:

- **989 документов** на английском языке;
- **496 документов** на русском языке.

Выполнены:

- распаковка архивов;
- дедупликация по SHA-256;
- извлечение текста;
- очистка текста;
- контроль качества;
- разметка тем.

Дополнительно размечены **9 тематических направлений** и **2 385 ключевых терминов** через TF-IDF.

## Архитектура

Архитектура решения построена как end-to-end конвейер: от исходных документов до ответа с цитатами.

1. Документы проходят инвентаризацию и нормализацию, то есть приводятся к единому виду хранения и взаимодействия с ними.
2. Из файлов извлекается текст.
3. Выполняется очистка текста.
4. Определяется язык и качество текста.
5. Документы разбиваются на фрагменты примерно по 400 токенов с сохранением информации о странице.
6. Для фрагментов строятся эмбеддинги `multilingual-e5` размерности 768.
7. Данные загружаются в Neo4j.
8. В Neo4j используются графовая модель и HNSW-векторный индекс.
9. Пользовательский запрос преобразуется в embedding.
10. Neo4j находит ближайшие чанки через векторный индекс.
11. Граф восстанавливает документ, страницу, соседние фрагменты и тематический контекст.
12. YandexGPT API формирует ответ строго по найденному контексту.

Модель YandexGPT не дообучалась и не тюнилась: она используется только как генератор финального ответа.

## Графовая модель

В проекте реализован граф на базе Neo4j. Сейчас он используется как основа GraphRAG-поиска: хранит документы, текстовые фрагменты, метаданные и связи с источниками.

Это позволяет:

- находить релевантные участки корпуса;
- переходить к исходным документам;
- восстанавливать страницу источника;
- формировать ответы с проверяемыми цитатами.

Графовая модель спроектирована с учетом дальнейшего развития. Текущий граф решает задачу поиска и навигации по корпусу, а следующим шагом может быть расширение до полноценной предметной карты знаний R&D: с материалами, процессами, оборудованием, экспериментами, параметрами, экспертами и связями между ними.

## Пользовательский интерфейс

Пользовательский интерфейс реализован как веб-приложение. В нем можно:

- ввести вопрос к корпусу;
- выбрать фильтры по году, теме, языку и типу документа;
- получить ответ;
- увидеть документы-источники;
- перейти к найденным фрагментам.

Система поддерживает сценарий, когда вопрос задан на одном языке, а источники используют другой. Например, пользователь может задать вопрос по-английски, а получить ответ на русском языке с опорой на англоязычные документы. Это возможно потому, что русские и английские тексты находятся в одном семантическом пространстве.

## Примеры проверенных запросов

Для демонстрации были проверены инженерные запросы, близкие к требованиям задания.

Например, на вопрос о температурах и давлении автоклавного выщелачивания сульфидных концентратов система выделяет температурные режимы и параметры экспериментов, указывая конкретные документы и страницы.

На вопрос о факторах, влияющих на мировое производство меди, система находит англоязычные отчеты и формирует русскоязычный ответ с источниками.

## Текущий статус

На текущий момент готова рабочая система полного цикла: вопрос пользователя превращается в ответ с цитатами.

Реализованы:

- веб-интерфейс;
- сервер для демонстрации;
- подготовленный корпус;
- графовая база;
- эмбеддинги;
- GraphRAG-поиск;
- воспроизводимые материалы: репозиторий, дамп базы, инструкция и демо.

## Дальнейшее развитие

Дальнейшее развитие проекта связано с расширением текущего графа документов до полноценной предметной карты знаний R&D.

Следующий слой - извлечение сущностей и связей поверх чанков:

- материалы;
- процессы;
- условия;
- параметры;
- оборудование;
- эксперименты;
- публикации;
- эксперты.

Это позволит выполнять более сложные запросы: например, искать все материалы по процессу при температуре выше 150 °C, сравнивать отечественную и зарубежную практику, находить противоречия между источниками и выявлять пробелы в исследованиях.

Также в план развития входит:

- гибридный поиск, объединяющий векторный и полнотекстовый подходы;
- расширение графовых фильтров;
- поддержка числовых диапазонов и единиц измерения;
- подсветка противоречий между источниками;
- экспертная корректировка графа;
- построение аналитических дашбордов по покрытию знаний.

## Результат

Главный результат проекта - доказанный рабочий контур GraphRAG для научно-технического корпуса.

Система уже помогает находить ответы за секунды вместо ручного поиска по документам, сохраняет проверяемость за счет ссылок на источники и имеет архитектурный запас для роста в полноценную карту знаний горно-металлургического R&D.
